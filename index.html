关于flex的总结

1、flex-grow 项目放大 默认值是0 即使子的所有的长度相加还是小于父的长度也不放大，也就是剩余空间存在也不缩小

2、flex-shrink 项目缩小 默认值是1 如果所有的子长度相加大于父的长度会缩小，也就是项目剩余的空间不存在会缩小适应整个项目，当0不缩小

3、flex-basic 项目默认最小宽度 项目本身的大小

4、flex auto 1 1 auto none 0 0 auto

js隐式转换总结

1、无论针对什么类型的数据（基本类型和对象类型）都会发生隐式转换的操作符有 + == 而 - * / 这三个操作符针对类型发生隐式转换，最终的转换结果只会是Number类型

2、隐式转换的顺序 Date类型的数据先调用toString这个方法，其他的都是先调用valueOf这个方法

3、valueOf针对不同类型返回的值类型（经过强制转换之后 new ….）

1、Number。String。Boolean返回的是值得强制转换原始数据
2、Date返回的是毫秒值
3、其他的返回的是自己本身

4、toString针对不同类型返回的值类型（经过强制转换之后new ….）

1、Number。String。Boolean返回的强制转换成对象之后的值
2、Date返回的是具体的时间
3、function返回的是函数的本身
4、Array返回的整个数组的值【1，2】=》 ‘1，2’
5、其他的返回的都是对象的类型都是调用 Object.prototype.toString.call

5、所有的隐式转换都会针对优先级来比较，有number的存在，所有的类型都会被转换成number，如果没有number的存在，有String的存在，那么其他的类型就会转成String

研究VDOM基础知识的普及

常用节点类型

节点名称 值 例子
node.element_node 1 一个 元素 节点，例如p、div
node.text_node 3 Element 或者 Attr 中实际的 文字
node.comment_node 8 一个 Comment 节点。

获取节点的对象 获取节点名称所对应的数字 nodeType就可以

  • createElement vs. createElementNS特性

createElement是创建一个HTML5DOM节点，而createElementNS是创建一个XML节点

  • Node.textContent与innerText与innerHTML区别

如果你设置了 textContent 属性, 任何的子节点会被移除及被指定的字符串的文本节点替换。使用textContent属性设置或返回指定节点及其子节点的文本内容。

以下是来自于MDN的介绍

  • 正如其名称，innerHTML 返回 HTML 文本。通常，为了在元素中检索或写入文本，人们使用innerHTML。但是，textContent通常具有更好的性能，因为文本不会被解析为HTML。此外，使用textContent可以防止 XSS 攻击。
  • textContent 会获取所有元素的内容，包括 script 和 style 元素，然而 innerText 不会。
  • innerText意识到样式，并且不会返回隐藏元素的文本，而textContent会。
  • 由于 innerText 受 CSS 样式的影响，它会触发重排（reflow），但textContent 不会。
  • 与 textContent 不同的是, 在 Internet Explorer (对于小于等于 IE11 的版本) 中对 innerText 进行修改， 不仅会移除当前元素的子节点，而且还会永久性地破坏所有后代文本节点（所以不可能再次将节点再次插入到任何其他元素或同一元素中）。
